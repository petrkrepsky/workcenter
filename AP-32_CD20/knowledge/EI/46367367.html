číselník InstrumentTransactionType

Do CSc. je potřeba zavést číselník InstrumentTransactionType.

zdroje dat

Číselník čerpá ze tří zdrojů dat:

  • eBroker - Typy pohybu
  • TA - Typy operace
  • Topas - Produkty

Datový obsah z eBroker je zřejmě velice stabilní (beze změn odhadem od r. 2011).

Datový obsah z TA a Topasu se zřejmě v čase mění.

Obsah číselníku proto definuji selectem.

zdrojový select

Obsah číselníku je dán tímto selectem:

-- select - zdroj dat pro CSC
-- ---------------------------------------------

with
  Zdroj_Topas as (
    select
        ID,
        Name,
        min(case when Lang_Code='CS' then Name_Lang end)                as Name_CS,
        min(case when Lang_Code='SK' then Name_Lang end)                as Name_SK,
        min(case when Lang_Code='EN' then Name_Lang end)                as Name_EN
      from (
        select
            x.IDProduct                                                 as ID,
            x.Name,
            y.Code                                                      as Lang_Code,
            topas_fo.Top_ITF_Pkg.GetProductDescription(x.IDProduct,y.IDLanguage)
                                                                        as Name_Lang
          from topas_fo.ITF_Product x
            cross join (
                -- číselník jazyků
                select
                    IDLanguage,
                    case Code when 'CZKL' then 'CS' else Code end       as Code
                  from topas_fo.ITF_Language
                  where Code in ('CZKL', 'SK', 'EN')
                ) y
        )
      group by ID, Name
    ),
  Zdroj_TA as (
    select IDOperationType as ID, Name,
        arbesta.TAITF_Operation_Pkg.GetText(Name,'cs') as Name_CS,
        arbesta.TAITF_Operation_Pkg.GetText(Name,'sk') as Name_SK,
        arbesta.TAITF_Operation_Pkg.GetText(Name,'en') as Name_EN
      from arbesta.TAITF_OperationType
    ),
  Zdroj_eBroker as (
    select
        x.ID,
        x.Apl_Kod                                                       as Name,
        y.Name_CS,
        y.Name_SK,
        y.Name_EN
      from ebroker.A_Typ_Pohybu x
        left outer join (
            -- texty v jazykových mutacích
            select ID_Objektu,
                min(case when AK_Jazyk='cs' then Text end)              as Name_CS,
                min(case when AK_Jazyk='sk' then Text end)              as Name_SK,
                min(case when AK_Jazyk='en' then Text end)              as Name_EN
              from ebroker.Text_V_Mutaci
              where Trida    ='a_typ_pohybu'
                and Vlastnost='nazev'
                and AK_Jazyk in ('cs', 'sk', 'en')
              group by ID_Objektu
            ) y
          on y.ID_Objektu=to_char(x.ID)
      where x.Apl_Kod is not null
    ),
  Zdroj_Vse as (
    select
        'topas'  ||'_'||to_char(ID)                                     as ID,
        Name, Name_CS, Name_SK, Name_EN
      from Zdroj_Topas
    union all
    select
        'ta'     ||'_'||to_char(ID)                                     as ID,
        Name, Name_CS, Name_SK, Name_EN
      from Zdroj_TA
    union all
    select
        'ebroker'||'_'||to_char(ID)                                     as ID,
        Name, Name_CS, Name_SK, Name_EN
      from Zdroj_eBroker
    )
-- účinný select
select *
  from Zdroj_Vse
  order by ID;

vzorek dat

Takto vypadá vzorek dat (Prod, 15.05.2023):

Vedle ID a Name ve třech jazykových mutacích uvádím ještě Name, které by se asi mohlo použít v případě, že hodnota v jazykové mutaci není k dispozici, jako například u topas_1003 a topas_1004.

upozornění - na různých prostředích se obsah číselníku liší

Upozorňuji, že na různých prostředích se obsah číselníku liší, leckdy zřejmě i podstatně.
