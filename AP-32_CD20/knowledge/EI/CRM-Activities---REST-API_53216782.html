Enterprise Integration : CRM Activities - REST API

Created by Kratochvíl Aleš, last modified on Dub 17, 2023

Požadavky na rekonciliaci

Samostatné API (oddělené od druhého API activities) sloužící pro rekonciliaci (a inicializaci) bude vystaveno na WSO2. Specifikace rozhraní je na stránce Integrační analýzy - open api.

Princip

Znovu zaslání pro určitého klienta zaznamenaných aktivit.


Aplikační služby

Jméno

Typ

activitiesReconciliation

REST API


Atributy volání

  • metody:
    • GET /activity/{activityType}
    • GET /activity/{activityType}/{activityId}
    • GET /client/{crmIdType}/{crmId}
    • GET /health (viz healthcheck endpoint)
  • header parametry:
    • X-Correlation-ID
    • traceparent
  • path parametry 
    • activityType je druh aktivity - např. schůzka, e-mail, telefonát, ...
    • activityId je identifikátor instance aktivity v CRM
    • crmIdType je druh identifikátoru klienta - např. AccountGUID nebo AccountKLID (kvůli bezpečnosti nelze využít pro RČ nebo IČ)
    • crmId je identifikátor instance klienta v CRM
  • query parametry (nepovinné):
    • klientId (AccountKLID) je číslo klienta (nemá smysl pro /client/{crmIdType}/{crmId})
    • filledWhenFrom (omezení výběru od - datum změny entity)
    • filledWhenTo (omezení výběru do - datum změny entity)

Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace.

Inicializace znamená:

  • zaslání všech nastavení v Activities
