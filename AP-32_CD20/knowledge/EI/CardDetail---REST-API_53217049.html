Požadavky na rekonciliaci


Specifikace rozhraní je na stránce Integrační analýzy - open api

Atributy volání

Na vstupu lze omezit konkrétní karetní detail pro který lze data zaslat.

Přehled metod a parametrů

  • metody:
    • GET /cardDetail"
  • header parametry (povinné):
    • X-Correlation-ID
    • traceparent
  • query parametry (nepovinné):
    • cardId
    • aPan
    • ownerCrmId
    • holderCrmId
    • masterAccountId
    • openDateFrom
    • cardExpiration (RRMM)

IMPLEMENTACE priorit query parametrů: Bude aplikován vždy pouze právě jeden query parametr. Pokud jich přijde v requestu více, ostatní parametry se zahodí, priorita je dána výše uvedením.


Požadavky na inicializaci

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. 

  • Do INIT/REKONCILAČNÍHO  topicu budou vloženy všechny účty, které jsou v historii BISQ dostupné. A zároveň se pošlou účty ve všech stavech s výjimkou stavu "A" (Authorized).
  • V jednotlivých událostech budou účty v aktuálním stavu, historie změn není evidována.
