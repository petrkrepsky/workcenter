Rozšíření request a response parametrů

Přidání header parametru traceparent

    • Specifikace požadavku:

      • Na rozhraní je požadováno přidat header parametr traceparent pro request i response.
      • Parametr je povinný typu string.
      • Parametr bude vyplňovat konzument rozhraní a bude se propagovat skrze všechny systémy, kterými volání prochází. Momentálně se neřeší jeho napojení do centrálního logování.
      • Hodnota traceparent v requestu je stejná jako v response.
      • Dokumentace: https://www.w3.org/TR/trace-context/
    traceparent: 
      description: "Tracing parameter according to W3C Trace Context standard."
      required: true
      schema: 
        maxLength: 55
        type: "string"
      example: "00-0af7651916cd43dd8448eb211c80319c-b7ad6b7169203331-01"

Definice rozhraní

Changelog - přidání do OpenAPI definice
    • Specifikace požadavku:
      • Při změně rozhraní do OpenAPI definice vložit vždy nový řádek s: datum, verze, autor, seznam změn.
      • Nedělat zpětně.
      • Pokud je OpenAPI definice generovaná a přidání changelogu nelze provést automaticky, je potřeba ho přidat ručně. Rozhraní by měla být dodávána s patřičnou dokumentací - ideálně changelog a manuál.
      • Příklad: # 2022-06-15 – 1.12.0 - Pavel Hrdlička - Added resource "/client", added query parameter "city" in "GET /userInfo"
Verzování - přidání do OpenAPI definice
    • Specifikace požadavku:
      • Do OpenAPI definice rozhraní je potřeba přidat verzi API.
      • Verze API by měla mít formát MAJOR.MINOR.PATCH, přičemž MINOR a PATCH není povinné (například 1.2.1 nebo 1.1).
      • Nová MAJOR verze se vydá jen v případě zpětně nekompatibilních změn (odstranění resource, přidání povinného parametru...).
      • Změny, které mají pouze informativní charakter (aktualizace kontaktů, doplnění OpenAPI definice, doplnění dokumentace), znamenají navýšení PATCH verze.
      • Verze API na WSO2 se řídí podle verze uvedené v OpenAPI definici rozhraní od BE systému.
      • Odkaz do metodiky: https://confluencedigi.jtfg.com/display/EI/REST+API
