Předmět a výstup rekonciliace

Rekonciliační API bude vystaveno ve WSO2. 

Pro BankNews se zachová stávající API vystavené na WSO2. Přiložený soubor je jím i inspirován

Rekonciliační API

parties_marketingList_Reconciliation_API.yaml

parties_BankNews_Reconciliation_API.yaml

Princip

Znovuzaslání výsledných změn všech entit nebo jedné datové entity z časového intervalu

Atributy volání

  • metody:
    • GET /marketingList/list
    • GET /marketingList/{marketingListId}/members
    • GET /BankNews/List
    • GET /bankNews/settings/{crmIdType}/{crmId}
  • header parametry (povinné):
    • X-Correlation-ID
  • query parametr (nepovinné):
    • from (pouze pro /marketingList)
    • to (pouze pro /marketingList)
    • marketingListId (pouze pro /marketingList/{marketingListId}/members)
    • mode

Předmět a výstup inicializace

Inicializace znamená:

  • zaslání všech marketingových listů v posledním stavu
  • zaslání všech nastavení bankNews

Existujíví API

API, která již existují a v době analýzi nepracují nijak s topicy. Budou upravea, aby splňovala podmínky pro začlenění do Enterprise integrace

Stávající popis pro integrační rozhraní je zde

Bank news

  • GET​/api​/v1​/BankNews​/list
    • Získání kolekce existujících rozesílek
    • bez vazby na klienta, jen seznam dostupných existujících rozesílek v crmku, které se dají použít
    • identifier se pak používá pro subscribe/unsubscribe 


  • GET/api/v1/BankNews/settings/{identifierType}/{identifier}
    • Získání nastavení rozesílek pro konkrétního klienta/kontakt
    • Otisk stávajícího nastavení na entitě Zprávy z banky


  • PATCH​/api​/v1​/BankNews​/{identifierType}​/{identifier}​/subscribe
    • Úprava nastavení rozesílek
    • Nastaví na Zprávách z banky odpovídající rozesílku/y = true


  • PATCH​/api​/v1​/BankNews​/{identifierType}​/{identifier}​/unsubscribe
    • Úprava nastavení rozesílek
    • Nastaví na Zprávách z banky odpovídající rozesílku/y = false
