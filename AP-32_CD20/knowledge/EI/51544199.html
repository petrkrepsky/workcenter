Implementace 

Řešení je postaveno na orchestrační platforme ESB Metada, která orchestruje API volání dvou systému:

1) ASB Sezam - zrušení vazby disponenta

2) CRM - zrušení vazby disponenta

Tato část dokumentace se zabývá implementaci pro zrušení uživatele v aplikaci ASB Sezam.

ASB Sezam je autentifikační/autorizační server pro DKCZ, který využívá OAuth 2.0 s OpenID Connect. 

Pro zrušení uživatele v Sezamu i s jeho bezpečnostními prostředky je nutné, aby ESB Metada provolala 2 API. První GET API vrací detail o uživateli, zde vybere hodnotu z odpovědi "version", kterou následně použijeme v DELETE metodě (Pole je povinné pro DELETE metodu, proto jej musíme získat prvního volání GET metody). Poslední metoda DELETE uživatele zruší i s bezpečnostními prostředky.

ESB Metada provádí toto volání pouze, když obdrží hodnotu "asbSezam": true z API /cancelClientDisponentBond. Jestliže je hodnota "asbSezam": false ESB Metada neprovede volání do ASB Sezam a pokračuje voláním CRM, v případě erroru Camunda.
