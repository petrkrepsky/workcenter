Enterprise Integration : CRM RealTimeDataAccess - REST API

Created by Kratochvíl Aleš, last modified on Dub 19, 2023

Požadavky na rekonciliaci

Samostatné API (oddělené od druhého API realTimeDataAccess) sloužící pro rekonciliaci (a inicializaci) bude vystaveno na WSO2. Specifikace rozhraní je na stránce Integrační analýzy - open api.

Princip

Znovu zaslání pro určitého klienta nastavení přístupu k real-time datům včetně vyplněných dotazníků z časového intervalu pro všechny či konkrétní burzu, případně znovu zaslání konkrétního vyplněného formuláře včetně nastavení z něho vycházejících.


Aplikační služby

Jméno

Typ

realTimeData

REST API


Atributy volání

  • metody:
    • GET /realTimeDataAccess
    • GET /realTimeDataAccess/{applicationId}
    • GET /health (viz healthcheck endpoint)
  • header parametry:
    • X-Correlation-ID
    • traceparent
  • path parametry 
    • applicationId je instance formuláře v CRM
  • query parametry (nepovinné a jen pro /realTimeDataAccess):
    • accountId (AccountGUID) je číslo účtu nebo klientId (AccountKLID) je číslo klienta
    • filledWhenFrom (omezení výběru od - datum změny entity)
    • filledWhenTo (omezení výběru do - datum změny entity)
    • marketFormId (tj. burza), pro kterou byl vyplněn dotazník

Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace.

Inicializace znamená:

  • zaslání všech nastavení v RealTimeDataAccess
