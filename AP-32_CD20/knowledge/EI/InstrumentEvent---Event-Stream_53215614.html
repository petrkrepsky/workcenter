Interface tabulky

eBroker vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu
  • Kontrolní

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Ve sloupci KEY klíč události = instrumentIds.ccpId.

Value

Přehled interface tabulek:

Kontrolní

  • CREATED - Čas vložení do interfacové tabulky.

  • TOTAL_EVENTS - Počet událostí v rekonciliaci, inicializaci nebo v asynchronní odpovědi.
  • X_CORRELATION_ID - Unikátní identifikátor volání, který je zaslán v hlavičkovém parametru rekonciliačního REST API, na vstupu initu nebo v požadavku na asynchronní odpověď.

Popis odběru dat

eBroker vkládá data do interface tabulek z nichž jsou vybírána Kafkou prostřednictvím JDBC konektoru.
Spouštěče událostí pro vkládání dat jednotlivých datových entit do interface tabulky, jejich frekvence a další aspekty jsou popsány v následující tabulce: Majer Karel  prosím o doplnění

  • vložení/smazání nebo změna relevantních hodnot v záznamu  centrum_cp.Cenny_Papir
  • vložení/smazání nebo změna relevantních hodnot v záznamu  centrum_cp.Udalost_CP
  • Cenny_Papir: řádově jednotky
  • Udalost_CP: řádově desítky
Velice záleží na procesu údržby databáze CP. Může se stát, že týden nic, může se stát, že za den vloží/aktualizují 1000 dluhopisů (mohou importovat data např. z Bloomberg), jejich kalendáře kupónů mohou mít desítky řádků (tj. desítky datových entit).

JSON schémata

JSON schémata obsahují nepovinný atribut X-Correlation-Id, který se plní hodnotu header parametru stejného jména z volání init/recon API.
