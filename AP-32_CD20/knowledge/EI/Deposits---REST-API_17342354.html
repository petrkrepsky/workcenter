Požadavky na rekonciliaci

Rekonciliační API bude vystaveno ve WSO2. Bude se jednat o stejné API jako pro entitu Accounts: Account - REST API


Atributy volání

Na vstupu omezujeme volání na konkrétní účet pro který lze transakce zaslat.

Přehled metod a parametrů:

  • metody:
    • GET /accounts/
  • header parametry (povinné):
    • X-Correlation-ID
  • query parametry:
    • accountProductId (nepovinné)
    • partyId (nepovinné)
    • je zasílán jeden z výše uvedených parametrů


Požadavky na inicializaci

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. 

  • Do INIT/REKONCILAČNÍHO  topicu budou vloženy všechny účty, které jsou v historii BISQ dostupné. S výjímkou těch, které jsou ve stavu "A".
  • V jednotlivých událostech budou účty v aktuálním stavu, historie změn není evidována.

Comments:

Dle mého názoru musí byt třetí query parametr accountId. 

Takže requeist musí obsahovat  jeden z parametru: partyId, accountId, accountProductId.

Důvodem je že, API pro založeni vrací accountId, jen přes něho je možná první rekonciliace pokud primární topik selže.,   

Na druhou stranu u RDA identifikátorem celku je accountProductId - musí vstupovat do rekonciliaci aby bylo možné zaslat cely cyklus prolongace .
