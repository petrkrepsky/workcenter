Interface tabulky

eBroker vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu
  • Kontrolní

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Ve sloupci KEY klíč události = ebrId.

Value

Východiska:

Rozdělení dat do interface tabulek vychází z několika kritérií:

  • Mít samostatnou tabulku pro přenosy s vysokým počtem událostí za den
  • Sdružovat do jedné interface tabulky události, které mají nízký počet událostí za den
  • Mít zvláštní interface tabulku pro kontrolní topic

Se zohledněním těchto pravidel je níže definována množina tabulek pro průběžný topic, init/recon topic a kontrolní topic.
