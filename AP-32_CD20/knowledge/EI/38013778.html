Interface tabulky

JTDOC vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu
  • Kontrolní

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Kontrolní topic

Klíč je definován na stránce Xerox - Připojovací analýza

Průběžný topic

V tuto chvíli se jako unikátní identifikátor používá confirmationId

Value

Data produktu ve formátu JSON.

Obsah těla je definován schématem "controllSchema_v1.json" na stránce Xerox - Připojovací analýza

Průběžný topic

Obsah těla je definován schématem uloženým na této stránce v kapitole JSON schéma. Volitelný atribut X-Correlation-ID se nevyplňuje.

  • KEY - vysvětleno výše
  • businessChangeDate = Datum změny datové entity.
  • BODY = JSON tělo zprávy ve formátu CLOB datové entity Confirmation

Inicializační/Rekonciličaní topic

Obsah těla je definován schématem uloženým na této stránce v kapitole JSON schéma. Oproti průběžnému topicu musí události v inicializačním/rekonciliačním topicu obsahovat atribut X-Correlation-ID, jehož hodnota je rovna hodnotě header parametru X-Correlation-ID z volání rekonciliačního API.

  • KEY - vysvětleno výše
  • businessChangeDate = Datum změny datové entity.
  • BODY = JSON tělo zprávy ve formátu CLOB datové entity Statement

Kontrolní topic

  • CREATED - Čas vložení do interfacové tabulky.

  • TOTAL_EVENTS - Počet událostí v rekonciliaci, inicializaci nebo v asynchronní odpovědi.
  • X_CORRELATION_ID - Unikátní identifikátor volání, který je zaslán v hlavičkovém parametru rekonciliačního REST API, na vstupu initu nebo v požadavku na asynchronní odpověď.
