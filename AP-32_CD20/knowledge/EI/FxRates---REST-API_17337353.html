Enterprise Integration : FxRates - REST API

Created by Joachymstál Vít, last modified on Říj 24, 2022

Předmět a výstup rekonciliace

Rekonciliační API bude vystaveno ve WSO2. Až se tak stane, některé detaily z této stránky budou odstraněny.

Specifikace rozhraní je na stránce Integrační analýzy - open api

Princip

Na vstupu lze omezit časový interval pro který se mají data poslat.

  • Časový interval (businessChangeDateStart/businessChangeDateEnd) je definovaný = znovuzaslání entit po poslední změně z časového intervalu, tedy pouze aktuální stavy entit
  • Časový interval (businessChangeDateStart/businessChangeDateEnd) není definovaný = znovuzaslání entit po poslední změně za poslední tři měsíce, tedy pouze aktuální stavy entit
  • Algoritmus je následující:
    • Není-li uvedeno businessChangeDateEnd, dosadí se aktuální datum a čas.
    • Není-li uvedeno businessChangeDateStart dosadí se businessChangeDateEnd minus 3 měsíce (včetně času).
  • Status deleted - je aktuální podoba datové entity. V rekonciliaci se datové entity ve stavu deleted přenášejí.

Východiska:

  • Mřížka na obou osách grafu odpovídá kroku 1 den.
  • Volitelně lze filtrovat podle časových intervalů (jeden, druhý nebo kombinace obou)
    • validityDateStart - validityDateEnd (v příkladu níže se při takové volbě intervalu pošlou příslušné datové entity označeny modře a spadající do tohoto intervalu)
    • businessChangeDateStart - businessChangeDateEnd (v příkladu níže se při takové volbě intervalu nepošle žádná entita)
  • Princip je ilustrován na příkladu vývoje kurzovního lístku ECB v čase
  • Modré jsou platné záznamy
  • Šedé jsou zneplatněné záznamy

Atributy volání

  • metody:
    • POST /fxRatesInstant/query/
    • POST /fxRatesDaily/query/
  • header parametry (povinné):
    • X-Correlation-ID
    • X-Request-ID
  • body parametry (nepovinné):
    • validityDateStart
    • validityDateEnd
    • businessChangeDateStart
    • businessChangeDateEnd
    • fxRateListId - specifikuje druh kurzovního lístku (například ecb|čnb|valuty|devizy_standard|devizy_premium)

Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. Zavoláním inicializace se rozumí vyvolní toku všech historických událostí od roku 2015.

Status deleted - je aktuální podoba datové entity. V inicializaci se datové entity ve stavu deleted nepřenášejí.

Atributy volání

  • povinné
    • X_Correlation_ID
  • nepovinné
    • Entity_Type
    • FX_Rate_List_ID
