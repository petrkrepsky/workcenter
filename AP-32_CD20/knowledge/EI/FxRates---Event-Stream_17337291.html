Interface tabulky

CCP vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Ve sloupci KEY je část složeného klíče. Jeho obsah pro jednotlivé datové entity:

  • FxRatesInstant" KEY = fxRateListId
  • FxRatesDaily: KEY = fxRateListId

Value

Průběžný topic

  • KEY - vysvětleno výše
  • businessChangeDate = Datum a čas zápisu kurzu datafeedem do databáze = businessChangeDate z těla JSONu
  • BODY = JSON tělo zprávy ve formátu CLOB datových entit:
    • FxRatesInstant
    • FxRatesDaily
  • LOCATION - bude vždy prázdná hodnota

    • FxRatesInstant
    • FxRatesDaily
  • LOCATION - bude vždy prázdná hodnota
  • X_CORRELATION_ID = Hodnota header parametru X-Correlation-ID z volání rekonciliačního API

  • CREATED - Čas vložení do interfacové tabulky.

  • TOTAL_EVENTS - Počet událostí v rekonciliaci, inicializaci nebo v asynchronní odpovědi.
  • X_CORRELATION_ID - Unikátní identifikátor volání, který je zaslán v hlavičkovém parametru rekonciliačního REST API, na vstupu initu nebo v požadavku na asynchronní odpověď.

Popis odběru dat

CCP vkládá data do interface tabulek z nichž jsou vybírána Kafkou prostřednictvím JDBC konektoru.
Spouštěče událostí pro vkládání dat jednotlivých datových entit do interface tabulky, jejich frekvence a další aspekty jsou popsány v následující tabulce:

Datová entitaSpouštěč událostiFrekvencePoznámka
FxRatesInstantuložení nových hodnotcca 1x denně v pracovních dnech, možná i vícekrát za den
  • Hodnoty obchodních kurzů měny se aktualizují z editoru.
  • Při uložení hodnot se do interface tabulky Kafka dostane nová sada dat (implementační poznámka od SiteOne: nutno analyzovat a implementovat, nejde udělat na db trigger)
FxRatesDailyuložení nových hodnotcca 1x denně v pracovních dnech, možná i vícekrát za den; změny kurzů ČNB a ECB budou různé události
  • Hodnoty se aktualizují importem dat pomocí datafeedu.
  • Při uložení hodnot se do interface tabulky Kafka dostane nová sada dat (implementační poznámka od SiteOne: nutno analyzovat a implementovat, nejde udělat na db trigger)

JSON schémata

JSON schémata FxRatesInstant a FxRatesDaily obsahují nepovinný atribut X-Correlation-ID, který se plní hodnotu header parametru stejného jména z volání init/recon API.

JSON Schéma je uloženo na stránce Integrační analýzy - schemata a formáty

Požadavky na integraci

  • počet partitions (8), pro číselník méně (1)
  • počet replik (3)
  • retention period (6 days)
