Popis publikace dat


Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY

SHAREPOINT publikuje data o účtech přímo do Průběžných případně rekoncilizačních topiců.

BID = documentId

BUSINESS_STATE = action

BCD (business change date) = timestamp

Vznik událostí

Spouštěčem je vytvoření, smazání nebo update dokumentu v SHAREPOINT. Zároveň existuje rekonciliační API.

JSHAREPOINT bude zasílat události ve všech stavech: create / update / delete

Filtrování událostí vyplývá z výše uvedeného, budou se zasílat události ve všech stavech.
