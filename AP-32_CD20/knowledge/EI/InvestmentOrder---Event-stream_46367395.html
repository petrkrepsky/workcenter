Interface tabulky

eBroker vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu
  • Kontrolní

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Ve sloupci KEY klíč události pro jednotlivé datové entity:

  • InvestmentNewOrder: KEY = orderIds.ebrokerId
  • InvestmentCancelOrder: KEY = orderIds.ebrokerId

Value

Přehled interface tabulek:

Kontrolní

  • CREATED - Čas vložení do interfacové tabulky.

  • X_CORRELATION_ID - Unikátní identifikátor volání, který je zaslán v hlavičkovém parametru rekonciliačního REST API, na vstupu initu nebo v požadavku na asynchronní odpověď.
