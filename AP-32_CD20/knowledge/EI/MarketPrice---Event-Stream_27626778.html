Interface tabulky

eBroker vytvoří, plní a spravuje tyto interface tabulky:

  • Pro čerpání dat do Průběžného topicu
  • Pro čerpání dat do Inicializačního/Rekonciličaního topicu
  • Kontrolní

Každá Kafka událost sestává z Key a Value:

  • Key do sloupce KEY
  • Value do sloupce BODY

Key

Ve sloupci KEY klíč události = instrumentId.

Value

Východiska:

Rozdělení dat do interface tabulek vychází z několika kritérií:

  • Mít pro OHLC samostatnou tabulku
  • Mít samostatnou tabulku pro přenosy s vysokým počtem událostí za den
  • Sdružovat do jedné interface tabulky události, které mají nízký počet událostí za den
  • Mít zvláštní interface tabulku pro kontrolní topic

Se zohledněním těchto pravidel je níže definována množina tabulek pro průběžný topic, init/recon topic a kontrolní topic.

problém s délkou sloupce Entity_Type v itf tabulkách

Zjistil jsem, že máme problém s délkou sloupce Entity_Type v itf tabulkách.

Sloupec je široký 30 znaků, ale při stanovování hodnot EntityType u InstrumentPrices jsme se rozjeli trochu více, než jsem očekával.

Konkrétně to vypadá takto:

entity_typelen
FxRatesDaily12
FxRatesInstant14
InstrumentMarketPriceDaily26
InstrumentMarketPriceInstant28
InstrumentMarketPriceInstantDelayed35
InstrumentValuePriceDaily25
InstrumentValuePriceInstant27
InstrumentValuePriceInstantDelayed34
InstrumentPerformanceMarketPriceDaily37
InstrumentPerformanceValuePriceDaily36
PortfolioOverview17

Navrhuji tedy změnit standard a sloupec Entity_Type definovat s maximální délkou 60 znaků. To by snad mohlo stačit.

Ptal jsem se kolegy programátora, za stranu SiteOne není problém nový standard zpětně uplatnit i na FxRates.

Nevím ale, co tyto změny znamenají pro implementaci přenosu v DK a pro konzumenty.
