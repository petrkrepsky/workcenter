Enterprise Integration : EnrollmentDigitalChannels - REST API

Created by Kratochvíl Aleš, last modified on Kvě 17, 2023

Požadavky na rekonciliaci

Samostatné API sloužící pro rekonciliaci (a inicializaci) bude vystaveno na WSO2. Specifikace rozhraní je na stránce Integrační analýzy - open api.

Princip

Znovu zaslání pro určitého klienta stavu zakládání v Digitálních kanálech.


Aplikační služby

Jméno

Typ

EnrollmentDigitalChannelsReconciliation

REST API


Atributy volání

  • metody:
    • GET /enrollmentDigitalChannels/{crmId}
    • GET /health (viz healthcheck endpoint)
  • header parametry:
    • X-Correlation-ID
    • traceparent
  • path parametry 
    • crmId je CRM identifikátor klienta (tzv. AccountKLID)
  • query parametry (nepovinné):
    • filledWhenFrom (omezení výběru od - datum změny entity)
    • filledWhenTo (omezení výběru do - datum změny entity)


Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace.

Inicializace znamená:

  • zaslání pro všechny klienty stavu zakládání v Digitálních kanálech
