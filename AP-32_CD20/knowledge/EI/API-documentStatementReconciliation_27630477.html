Enterprise Integration : API-documentStatementReconciliation

Created by Havlena David, last modified by Rak Filip on Dub 21, 2023

Předmět a výstup rekonciliace

Rekonciliační API bude vystaveno ve WSO2.

Specifikace rozhraní je na stránce Integrační analýzy - open api

Budou vystaveny dvě samostatné API a dva samostatné  End pointy pro DK CZ a DK SK, pro každý Digitální kanál je určeno samostatné API

Atributy volání

  • metody:
    • GET /statements
  • header parametry (povinné):
    • X-Correlation-ID
    • traceparent
  • query parametr (povinné):
    • locations
  • query parametr (nepovinné):
    • statementId
    • crmid
    • quaestorId
    • topasId
    • statementTypeCodeId
    • businessChangeDateStart
    • businessChangeDateEnd

Princip

buď znovu zaslání jedné datové entity dle identifikátoru (pouze jeden identifikátor ve zprávě)

  • statementId

bez časového intervalu

nebo znovu zaslání posledních změn entit 

  • crmid
  • quaestorId
  • topasId
  • statementTypeCodeId

 z časovým intervalem

  • businessChangeDateStart
  • businessChangeDateEnd

Pokud nepřijde v požadavku žádný z nepovinných atributů odpověď je 400 - Bad Request.

Pokud nepřijde v požadavku businessChangeDateStart/end bude se chovat rekonciliace chovat stejně jako inicializace, omezením poskytnutí dat - aktuální Time -90 dní.   

Pořadí kontroly atributů při plnění topicu je dáno prioritou (sloupec "priorita")

Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace.

Inicializace znamená:

  • zaslání metadat ke stažení požadovaného výpisu
