Enterprise Integration : PortfolioValuationDaily - REST API

Created by Joachymstál Vít, last modified on Bře 01, 2023

Rekonciliační API bude vystaveno ve WSO2. Až se tak stane, některé detaily z této stránky budou odstraněny.

Specifikace rozhraní je na stránce Integrační analýzy - open api

Předmět a výstup rekonciliace

Princip

Na vstupu lze omezit časový interval pro který se mají data poslat.

  • Časový interval (BCD) je definovaný = znovuzaslání entit po poslední změně z časového intervalu, tedy pouze aktuální stavy entit
  • Časový interval (BCD) není definovaný = znovuzaslání entit po poslední změně za poslední tři měsíce, tedy pouze aktuální stavy entit
  • V rekonciliaci se datové entity ve stavu deleted přenášejí.
  • Prázdná (nulová) portfolia:
    • Entita se posílá, pokud jde o její poslední změnu a okamžik vyprázdnění spadá do intervalu recon; stav entity je aktivní.

Atributy volání

  • metoda:
    • POST /portfolioValuationDaily/query/
  • header parametry (povinné):
    • X-Correlation-ID
    • X-Request-ID
  • query parametr (povinný):
    • location
  • body parametry (nepovinné):
    • businessChangeDateStart
    • businessChangeDateEnd
    • validityDateStart
    • validityDateEnd
    • portfolioIds
      • ebrokerId
    • owner
      • crmId

Předmět a výstup inicializace

Princip

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. Zavoláním inicializace se rozumí vyvolání toku historických událostí od validityDate 1.1.2015.

V inicializaci se datové entity ve stavu deleted nepřenášejí.

Prázdná (nulová) portfolia:

  • Entita se posílá; stav entity je aktivní.

Atributy volání

  • povinné
    • X_Correlation_ID
  • nepovinné
    • location
