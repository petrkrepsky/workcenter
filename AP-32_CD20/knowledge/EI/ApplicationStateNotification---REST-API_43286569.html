Enterprise Integration : ApplicationStateNotification - REST API

Created by Joachymstál Vít, last modified by Kratochvíl Aleš on Kvě 23, 2023

Předmět a výstup rekonciliace

Rekonciliační API bude vystaveno ve WSO2 a zastřešeno orchestrační službou v Metadě. Orchestrace zajistí, aby byl rekonciliační požadavek předán na všechny backendy (Camunda a CRM). Všechny backendy a orchestrační služba dodržují stejné rozhraní, které je definováno níže.

Specifikace rozhraní je na stránce Integrační analýzy - open api

Princip

Na vstupu lze omezit časový interval pro který se mají data poslat.

  • Časový interval je definovaný = znovuzaslání výsledných změn entit z časového intervalu
  • Časový interval není definovaný = znovuzaslání výsledných změn entit za poslední tři měsíce domluvit

Atributy volání

  • metody:
    • POST /applicationStateNotification/query
  • header parametry (povinné):
    • X-Correlation-ID
    • traceparent
  • body parametry (nepovinné):
    • businessChangeDateStart
    • businessChangeDateEnd
    • applicationConsumerId (ponovu instance žádosti v DK; volání s hodnotou crmApplicationId již není možné - viz ChR DSWF-4904)
    • applicationState

Předmět a výstup inicializace

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. Zavoláním inicializačního API se rozumí vyvolání všech událostí. Důležité je inicializovat ze všech (momentálně Camunda a CRM) dotčených backendů.

Atributy volání

  • povinné
    • X-Correlation-ID
