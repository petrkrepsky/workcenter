Enterprise Integration : Closing Balance - REST API

Created by Hrdlička Pavel, last modified on Úno 06, 2023

Požadavky na rekonciliaci

Specifikace rozhraní je na stránce Integrační analýzy - open api

Atributy volání

Na vstupu lze omezit konkrétní účet, pro který lze zůstatek zaslat.

Přehled metod a parametrů

  • metody:
    • GET /closingBalance/
    • GET /closingBalance/{date} (parametr "date" je ve formátu date, tedy YYYY-MM-DD)
      • doplnění info (6.2.2023): Jedná se o všechny události, u kterých došlo v daném datu ke změně.
  • header parametry (povinné):
    • X-Correlation-ID
  • query parametr:
    • pro GET /closingBalance/
      • accountId - (accountId z BISQ) je povinné právě jedno accountId nebo crmId - kontroluje BISQ implementačně
      • crmId - je povinné právě jedno accountId nebo crmId - kontroluje BISQ implementačně
      • dateFrom (povinné) - je ve formátu date, tedy YYYY-MM-DD

Požadavky na inicializaci

Inicializační API může, ale také nemusí, být REST služba. Záleží na konkrétní realizaci preferované vlastníkem aplikace. 

  • Výsledkem inicializace je naplnění INIT/REKONCILAČNÍHO topicu událostmi s informacemi o účetním zůstatku.
  • Budou se zasílat zůstatky pro všechny účty s výjimkou účtů ve stavu "A" (Authorized).
  • DK/SK i DK/CZ - nezasílat uzavřené účty před "euro konverzí".
